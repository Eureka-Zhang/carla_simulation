# Phase 1: 自然驾驶数据采集实验手册

## 实验概述

| 项目 | 内容 |
|------|------|
| **实验目的** | 采集被试自然驾驶风格数据，用于训练个性化跟驰模型 |
| **实验时长** | 约 10 分钟 |
| **被试角色** | 驾驶员 |
| **控制方式** | 驾驶舱硬件 |

---

## 实验前准备

### 1. 系统启动检查清单

```
□ CARLA 服务器已启动
□ 驾驶舱硬件已连接 (IP: 192.168.0.20, 端口: 3232)
□ 显示器已正确配置
□ VNC 远程监控已就绪（如需要）
```

### 2. 启动 CARLA 服务器

```bash
# 进入 CARLA 目录
cd /path/to/carla

# 启动服务器（根据实际路径）
./CarlaUE4.sh
```

### 3. 启动实验程序

```bash
cd ~/carla_simulation/car_following_experiment

# 驾驶舱模式
python car_following_experiment.py --cabin --map Town04

# 如需指定显示器
python car_following_experiment.py --cabin --display 0 --fullscreen
```

## 实验流程

### 步骤 1: 被试准备 (约 2 分钟)

1. **被试入座**
   - 调整座椅位置
   - 调整方向盘位置
   - 确认油门/刹车踏板舒适

2. **指导语**（向被试朗读）
   
   > "这是一个跟车驾驶任务。您前方会有一辆蓝色汽车，请跟随它行驶。
   > 
   > **请按照您平时最自然、最习惯的方式驾驶。**
   > 
   > 没有对错之分，我们只是想了解您的驾驶习惯。
   > 
   > 前车会有一些速度变化，请您自然应对即可。
   > 
   > 整个过程大约 10 分钟，准备好后请告诉我。"

3. **确认被试理解任务**

### 步骤 2: 热身练习 (约 1 分钟)

1. 让被试先熟悉操控
2. **不要开始正式数据采集**
3. 确认被试操控流畅后进入下一步

### 步骤 3: 正式实验 (约 10 分钟)

1. **重启场景**
   - 按 `Backspace` 重置车辆位置

2. **切换前车行为模式**
   - 按 `F3` 切换至 **"fixed (固定曲线)"** 模式
   - 屏幕会显示通知确认

3. **开始数据采集**
   - 按 `F1` 开始采集
   - 屏幕显示 "数据采集已开始"
   - 左上角会显示录制帧数

4. **提示被试开始**
   > "好的，现在开始，请自然驾驶。"

5. **实验进行中**
   - 主试保持安静，不干扰被试
   - 监控数据采集状态
   - 注意被试是否有异常

6. **实验结束**
   - 10 分钟后，按 `F1` 停止采集
   - 屏幕显示采集统计信息

### 步骤 4: 实验后 (约 1 分钟)

1. 感谢被试配合
2. 询问被试感受（非正式）
3. 记录任何异常情况

## 数据说明

### 数据保存位置

```
./experiment_data/
└── YYYYMMDD_HHMMSS/
    ├── driving_data.csv        # 主数据文件
    └── driving_data_metadata.json  # 元数据
```

### 数据字段说明

| 字段 | 单位 | 说明 |
|------|------|------|
| `timestamp` | 秒 | 相对时间戳 |
| `frame` | - | 帧序号 |
| `ego_speed` | m/s | 自车速度 |
| `ego_acceleration` | m/s² | 自车加速度 |
| `ego_jerk` | m/s³ | 自车加加速度 |
| `lead_speed` | m/s | 前车实际速度 |
| `lead_acceleration` | m/s² | 前车加速度 |
| `lead_target_speed` | m/s | 前车目标速度 |
| `distance_headway` | m | 车头间距 |
| `time_headway` | s | 时距 (THW) |
| `relative_speed` | m/s | 相对速度 (正=接近) |
| `ttc` | s | 碰撞时间 (TTC) |
| `throttle` | 0-1 | 油门开度 |
| `brake` | 0-1 | 制动强度 |
| `longitudinal_control` | -1~1 | 纵向控制信号 |
| `gear` | - | 当前挡位 |
| `lead_behavior_mode` | - | 前车行为模式 |

### 用于 IL 训练的关键字段

**状态空间 (State):**
- `ego_speed`
- `distance_headway`
- `relative_speed`
- `ego_acceleration` (可选)

**动作空间 (Action):**
- `longitudinal_control` (油门-制动合成信号)
- 或分别使用 `throttle` 和 `brake`

## 常见问题处理

### Q1: 被试发生碰撞怎么办？

1. 按 `Backspace` 重启场景
2. 记录碰撞时间点
3. 重新开始数据采集
4. 碰撞前的数据仍可使用（需在分析时截断）

### Q2: 数据采集中断怎么办？

1. 检查程序是否崩溃
2. 已保存的数据仍然有效
3. 重新启动程序，继续采集
4. 在元数据中记录中断情况

### Q3: 前车速度没有变化？

检查是否切换到了 "fixed" 模式：
- 按 `F3` 切换模式
- 确认屏幕显示 "前车行为: fixed"

### Q4: 驾驶舱无响应？

1. 检查 UDP 连接 (192.168.0.20:3232)
2. 检查驾驶舱电源
3. 重启实验程序

## 数据质量检查

采集完成后，快速检查数据质量：

```bash
cd ~/carla_simulation/car_following_experiment

# 查看数据文件
ls -la experiment_data/

# 检查数据行数（应约 18000 行，10分钟 × 30fps）
wc -l experiment_data/YYYYMMDD_HHMMSS/driving_data.csv

# 查看数据前几行
head -20 experiment_data/YYYYMMDD_HHMMSS/driving_data.csv
```

### 数据质量标准

| 指标 | 合格标准 |
|------|----------|
| 数据时长 | ≥ 9 分钟 |
| 数据帧数 | ≥ 16000 帧 |
| 碰撞次数 | ≤ 2 次 |
| 数据完整性 | 无大段缺失 |
